## Definition
学术工作流搭建（Academic Workflow Setup）

## Key Points

- 完整流程设计：从文献收集（Zotero）、笔记分析（NotebookLM）、知识沉淀（Obsidian）到代码实现（Cursor）的端到端工作流
- 工具选择标准：统一性（课题组标准化）、协同性（工具间数据流转）、可扩展性（适应未来需求）
- 工程习惯培养：材料用 Markdown 保存、代码可复现、版本管理（Git）、依赖管理（requirements.txt）
- 环境配置规范：Python 虚拟环境（venv）、项目结构标准化、脚本可从命令行运行
- 文档规范：README.md 项目说明、学习记录（notes/目录）、会议纪要、周报的 Markdown 化
- 效率优化：通过工具链集成减少手动搬运，将精力集中在核心科研问题上
